# TTT-Dialect: 노인·방언 화자 개인 적응 음성 인식 시스템
## 프로젝트 기획서

---

## 1. 프로젝트 개요

### 한 줄 정의
> 노인이 말하면 방언을 자동 감지하고, 그 사람 목소리에 실시간 적응해서  
> 기존 상용 STT보다 정확하게 인식하는 **3단계 개인화 음성 인식 파이프라인**

### 배경 및 문제 정의

기존 STT 서비스(네이버 클로바, 카카오 음성)는 표준어·청년층 기준으로 최적화되어 있다.  
노인 화자는 아래 세 가지 특성이 겹쳐 인식률이 40~50%대로 급락한다.

| 특성 | 설명 |
|------|------|
| 느린 발화 속도 | 표준 모델이 학습한 속도와 다름 |
| 발음 불명확 | 치아 문제, 노화로 인한 조음 변화 |
| 지역 방언 | 어휘·발음·억양이 표준어와 상이 |

이로 인해 키오스크 공포증, 콜센터 포기 등 디지털 소외가 심화되고 있다.

---

## 2. 핵심 기술

### 2-1. 방언 자동 감지 (Dialect Identifier) — 신규 추가
사용자가 말하면 어느 지역 방언인지 자동 분류.  
기존 연구들은 사용자가 방언을 직접 선택해야 했음.

```
입력: 음성 5~10초
출력: 경상 / 전라 / 충청 / 강원 / 제주 / 서울 + 신뢰도(%)
모델: Whisper 인코더 → 6-class 분류기
```

### 2-2. 방언별 특화 모델 (Dialect-Specific Fine-tuning)
6개 방언 각각에 특화된 Whisper 파인튜닝.  
Base 모델 1개 + 방언별 상위 레이어 가중치만 저장 (용량 효율).

```
Base Whisper
    ├── 경상도 레이어 가중치
    ├── 전라도 레이어 가중치
    ├── 충청도 레이어 가중치
    ├── 강원도 레이어 가중치
    ├── 제주도 레이어 가중치
    └── 표준어 레이어 가중치
```

### 2-3. TTT 개인 적응 (Test-Time Training) — 핵심 기술
추론(사용) 시점에 해당 사용자 발화로 모델을 실시간 업데이트.  
전체 파라미터의 ~8%(상위 2개 레이어)만 업데이트 → 빠른 적응, 과적합 방지.

```
캘리브레이션 흐름:
1. 방언 자동 감지
2. 해당 방언 특화 모델 로드
3. 사용자가 20문장 읽음 (약 3분)
4. 상위 2개 인코더 레이어 30 step 업데이트
5. 이후 이 사람 전용 모델로 인식

지속 적응:
- 인식 결과가 틀렸을 때 수정 → 즉시 추가 학습
- 사용할수록 정확도 향상
```

## 3. 기존 연구 대비 차별화

| 비교 항목 | 기존 연구 | 본 프로젝트 |
|-----------|-----------|-------------|
| 방언 처리 | 방언 분류만 하거나, 수동 선택 | **자동 감지 후 특화 모델 자동 전환** |
| TTT 적용 | 영어·이미지 도메인 | **한국어 노인·방언 최초 적용** |
| 적응 단위 | 방언 그룹 단위 fine-tuning | **개인 단위 실시간 적응** |
| 파이프라인 | 단일 단계 | **감지 → 특화 → 개인화 3단계** |
| 검증 환경 | 실험실 데이터 | **키오스크 실사용 환경 시뮬레이션** |

### 논문 기여 요약
```
Contribution 1: 방언 자동 감지 + TTT 개인화 end-to-end 파이프라인 최초 제안
Contribution 2: 한국어 노인 음성 도메인에 TTT 적용 및 효과 검증
Contribution 3: 방언별·연령대별 TTT 적응 효과 정량 분석
Contribution 4: 키오스크 실사용 환경에서의 성능 비교 (클로바·카카오 대비)
```

## 4. 시스템 구성

### 전체 파이프라인

```
[사용자 발화]
      ↓
[방언 자동 감지]  ← Whisper 인코더 + 6-class 분류기
      ↓
[방언 특화 모델 로드]  ← 해당 방언 레이어 가중치 적용
      ↓
[TTT 캘리브레이션]  ← 20문장, 30 step, 상위 2레이어만
      ↓
[개인화 인식]  ← 이 사람 전용 모델
      ↓
[지속 적응]  ← 수정 피드백 → 추가 학습
      ↓
[출력]
 - 인식 텍스트
 - WER 전후 비교
 - 방언별 성능 히트맵
```

### 사용 라이브러리

| 역할 | 도구 |
|------|------|
| 기반 STT | openai-whisper |
| 방언 분류기 | transformers + 커스텀 분류 헤드 |
| TTT 구현 | transformers + 커스텀 학습 루프 |
| 음성 처리 | librosa, torchaudio |
| 데이터 | AI Hub (방언·노인 음성) |
| 평가 | jiwer (WER/CER) |
| 데모 앱 | streamlit + plotly |

## 5. 데이터 전략

### 5-1. AI Hub 데이터 (메인 데이터)

| 데이터셋 | 내용 | 신청 방법 |
|----------|------|-----------|
| 한국어 방언 발화 | 경상/전라/충청/강원/제주 방언 | aihub.or.kr → 학술 신청 (무료, 1~3일) |
| 노인 음성 데이터 | 60세 이상 발화 | 동일 |

### 5-2. 즉시 사용 가능한 데이터 (AI Hub 승인 대기 중)
- **Common Voice 한국어** (HuggingFace, 즉시 다운로드)
- AI Hub 승인 전 파이프라인 검증용으로 사용

### 5-3. 데이터 증강 전략
노인 음성 특성을 반영한 인위적 증강:

```python
# 느린 발화 시뮬레이션 (time stretch, rate 0.85~1.0)
# 음역대 차이 시뮬레이션 (pitch shift, ±1.5 steps)
# 키오스크 환경 잡음 추가 (background noise)
```

## 6. 데모 앱 구성 (발표용 핵심)

### 화면 1: 방언 감지
```
[마이크에 대고 말함]
→ 한국 지도에서 해당 지역 빛남
→ "경상도 방언 감지됨 (신뢰도 87%)"
→ "경상도 특화 모델로 전환합니다"
```

### 화면 2: 실시간 TTT 적응
```
[문장 읽을 때마다]
→ WER 수치가 실시간으로 내려가는 라이브 그래프
→ "지금 AI가 당신 목소리를 학습하고 있습니다"
→ 20문장 완료 → "적응 완료! WER 52% → 29%"
```

### 화면 3: 기존 STT 비교 (발표 핵심 장면)
```
┌──────────────────────────────────────────────┐
│  네이버 클로바  │   카카오 음성  │  본 시스템  │
├──────────────────────────────────────────────┤
│ "오늘 날씨가   │ "오늘 날씨가  │ "오늘 날씨가 │
│  조개네요"     │  조재네요"    │  좋네요"     │
│      ❌        │      ❌       │      ✅      │
└──────────────────────────────────────────────┘
```

### 화면 4: 성능 대시보드
```
- TTT 전후 WER 비교 막대 그래프
- 방언별 성능 향상 히트맵 (6방언 × 연령대)
- 개인 적응 이력 (사용할수록 올라가는 정확도 곡선)
- 기존 STT 대비 개선율 비교표
```

## 7. 평가 지표

| 지표 | 설명 | 목표 |
|------|------|------|
| WER (단어 오류율) | 낮을수록 좋음 | TTT 후 30% 이하 |
| CER (글자 오류율) | 낮을수록 좋음 | 참고 지표 |
| 방언 감지 정확도 | 6개 방언 분류 정확도 | 85% 이상 |
| 적응 속도 | 캘리브레이션 소요 시간 | 3분 이내 |
| 개선율 | TTT 전 대비 WER 감소 | 20%p 이상 |

### 비교 대상
- Whisper Base (파인튜닝 없음)
- Whisper Fine-tuned (TTT 없음)
- **본 시스템** (방언 감지 + TTT)
- 네이버 클로바 API
- 카카오 음성 API

## 8. 개발 타임라인

### Phase 0 — 지금 당장
```
□ AI Hub 방언 데이터 신청
□ AI Hub 노인 음성 데이터 신청
□ Common Voice 한국어 다운로드 (파이프라인 검증용)
□ 기존 코드 검토 및 구조 확정
```

### Phase 1 — 1주차: 데이터 및 베이스라인
```
□ Common Voice로 Whisper 파인튜닝 파이프라인 검증
□ 방언 분류기 프로토타입 구현
□ TTT 적응 기본 동작 확인
□ WER 베이스라인 측정
```

### Phase 2 — 2주차: 핵심 모델
```
□ AI Hub 데이터 전처리 완료 (승인 시)
□ 방언별 Whisper 파인튜닝 (6개)
□ 방언 감지기 학습 및 평가
□ TTT 레이어 튜닝 (학습률, 스텝 수 최적화)
```

### Phase 3 — 3주차: 통합 및 검증
```
□ 3단계 파이프라인 통합 (감지 → 특화 → TTT)
□ 전체 WER 측정 (방언별·연령대별)
□ 클로바·카카오 API 비교 실험
□ 버그 수정 및 안정화
```

### Phase 4 — 4주차: 데모 앱
```
□ 방언 감지 화면 (지도 시각화)
□ 실시간 TTT 적응 화면 (라이브 WER 그래프)
□ 기존 STT 비교 화면
□ 성능 대시보드
□ 발표용 시나리오 리허설
```

### Phase 5 — 5주차: 마무리
```
□ 실험 결과 정리
□ 발표 자료 제작
□ 논문 초안 작성 (선택)
□ 최종 데모 리허설
```

## 9. 예상 결과

### 정량적 목표

| 모델 | 노인 음성 WER | 방언 포함 WER |
|------|--------------|--------------|
| 네이버 클로바 | ~52% | ~63% |
| Whisper Base | ~61% | ~72% |
| Whisper Fine-tuned | ~38% | ~45% |
| **본 시스템 (목표)** | **~25%** | **~30%** |

### 정성적 목표
- 발표장 라이브 데모 성공
- "방언 말하면 지역 자동 감지" 시연
- 청중 직접 참여 (마이크로 말해보기)
